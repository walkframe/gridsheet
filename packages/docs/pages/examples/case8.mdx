### Batched Data Generation with `setTimeout`

Generating 10,000 × 100 = 1,000,000 cell definitions synchronously would block the main thread and freeze the UI. The solution is to split the work into batches and yield control back to the browser between each batch using `setTimeout(..., 0)`.

```tsx
const batchSize = 1000;
const totalBatches = Math.ceil(rows / batchSize);

const generateBatch = (batchIndex: number) => {
  const startRow = batchIndex * batchSize + 1;
  const endRow = Math.min(startRow + batchSize - 1, rows);

  for (let row = startRow; row <= endRow; row++) {
    for (let col = 1; col <= cols; col++) {
      cells[address] = { value, style };
    }
  }

  setProgress(Math.round(((batchIndex + 1) / totalBatches) * 100));

  if (batchIndex + 1 < totalBatches) {
    setTimeout(() => generateBatch(batchIndex + 1), 0); // yield to browser
  } else {
    setIsLoading(false);
  }
};

generateBatch(0);
```

Calling `setProgress` after each batch keeps the progress bar reactive without adding extra overhead.

### Showing a Progress Bar During Generation

Use a `loading` state and a `progress` percentage to display a progress bar. The grid itself is not mounted until generation completes, preventing `initialCells` from being passed before it is ready.

```tsx
const [isLoading, setIsLoading] = React.useState(true);
const [progress, setProgress] = React.useState(0);

if (isLoading) {
  return (
    <div>
      <div style={{ width: `${progress}%`, height: '20px', backgroundColor: '#3b82f6' }} />
      <span>{progress}% Complete</span>
    </div>
  );
}

return <GridSheet book={book} initialCells={initialCells} ... />;
```

### Multi-letter Column Address Generation

Excel-style column addresses beyond `Z` (i.e., `AA`, `AB`, ...) require a custom converter. With 100 columns this covers through `CV`.

```tsx
const getColumnLetter = (colNum: number): string => {
  let result = '';
  while (colNum > 0) {
    colNum--;
    result = String.fromCharCode(65 + (colNum % 26)) + result;
    colNum = Math.floor(colNum / 26);
  }
  return result;
};
// getColumnLetter(1)  → 'A'
// getColumnLetter(27) → 'AA'
```

### `buildInitialCellsFromOrigin` vs Flat `cells` Object

This demo builds a plain `{ [address: string]: CellType }` object directly and passes it to `GridSheet` as `initialCells`. This avoids calling `buildInitialCells` on an already-generated flat structure and removes the overhead of the matrix-to-address conversion that `buildInitialCells` performs internally.

```tsx
const [initialCells, setInitialCells] = React.useState<{ [address: string]: any }>({
  defaultCol: { width: 80 },
  defaultRow: { height: 20 },
});

React.useEffect(() => {
  const cells = generateLargeDataset();
  setInitialCells(cells);
}, [generateLargeDataset]);
```

Setting `defaultCol` and `defaultRow` in the initial state means the grid renders immediately with reasonable dimensions, even while data generation is in progress.

### `sheetResize: 'both'` for Resizable Grid

The `sheetResize` option exposes a resize handle so users can adjust the grid dimensions interactively — useful when exploring wide or tall datasets at different viewport sizes.

```tsx
options={{
  sheetHeight: 600,
  sheetWidth: Math.min(1200, window.innerWidth - 60),
  sheetResize: 'both',
}}
```

### Applying Row Stripes with CSS

GridSheet automatically adds `gs-row-odd` and `gs-row-even` classes to each `<tr>` element based on the row's position (`y`). You can use these classes in a `<style>` tag to apply alternating row background colors without touching cell data:

```tsx
<>
  <GridSheet ... />
  <style>{`
    .gs-row-odd .gs-cell  { background-color: #ffffff; }
    .gs-row-even .gs-cell { background-color: #f0f4f8; }
  `}</style>
</>
```

Because these are CSS rules (not inline styles), any `style.backgroundColor` set directly on a cell object will take priority and override the stripe color for that specific cell.
